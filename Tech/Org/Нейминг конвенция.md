

## Правила нейминга для уровней:

| Asset Type         | Prefix | Suffix    | Sample                    | Description                                              |
| ------------------ | ------ | --------- | ------------------------- | -------------------------------------------------------- |
| Level (Persistent) | `L_`     | `_P`        | `L_Location_01_P`           | Базовый левел который содержит гейммод и логику          |
| Level (Audio)      | `SubL_`  | `_Audio`    | `SubL_Location_01_Audio`    | Саблевел который включает в себя все звуковые акторы     |
| Level (Lighting)   | `SubL_`  | `_Lighting` | `SubL_Location_01_Lighting` | Саблевел который содержит информацию о свете и освещении |
| Level (Geometry)   | `SubL_`  | `_Geo`      | `SubL_Location_01_Geo`      | Саблевел с геометрией локации                            |
| Level (Gameplay)   | `SubL_`  | `_Gameplay` | `SubL_Location_01_Gameplay` | Саблевел который содержит все акторы с логикой           |

Хорошей практикой является возможность хранить различные типы акторов в разных саблевелах. Persistent уровень является управляющим родителем, который отвечает за гейм мод и порядок загрузки саблевелов.  
Использование Lighting саблевела позволит создать различные сценарии освещения и переключать их на лету.

Если у текстуры предполагается наличие порядкового номера, ставим его перед суффиксом:

```
T_Floor_01_D
```

Также, если мы упаковываем текстуры по **RGBa** каналам, то нужно придерживаться правила, что суффикс будет содержать все суффиксы упаковываемых текстур в порядке упаковки.

Например:

|   |   |
|---|---|
|Текстуры которые мы упаковываем|Финальный результат|
|T_Floor_O|T_Floor_ORM|
|T_Floor_R|
|T_Floor_M|

Настоятельные рекомендации:

- Текстуры масок (Mask) желательно упаковывать только с масками или сохранять без упаковки
    
- Избегать упаковки в Alpha канал, так как сильно раздувает вес текстуры.
    
- При упаковке PBR текстур, необходимо придерживаться общего стандарта и упаковывать в ORM порядке.
